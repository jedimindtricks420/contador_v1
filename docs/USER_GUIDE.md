# Инструкция пользователя

## 1. Авторизация
При входе в систему браузер запросит логин и пароль. Это базовая HTTP-авторизация для защиты ваших финансовых данных.

## 2. Дашборд
Главная страница отображает:
- **Выручка:** Приход по счету 9010.
- **Расходы:** Все затраты по счетам 94*.
- **Чистая прибыль:** Разница между выручкой и расходами.
- **Деньги в банке:** Текущий остаток на счете 5110.
- **Дебиторка:** Задолженность клиентов перед вами (счет 4010).

## 3. Журнал операций
Это основной инструмент ввода данных.
- **Проведение операции:** Выберите дату, счета дебета и кредита, введите сумму и описание.
- **Период:** Указывается в формате `ММ.ГГГГ` (например, `03.2026`).
- **Удаление:** Операции не удаляются физически, а помечаются как удаленные (мягкое удаление), чтобы сохранить историю.

> **Важно:** Вы не сможете создать или удалить операцию за дату, которая входит в "Закрытый период".

## 4. Оборотно-сальдовая ведомость (ОСВ)
- Показывает обороты за выбранный месяц и итоговое сальдо.
- **Drill-down:** Нажмите на любую строку (счет), чтобы увидеть список всех проводок, сформировавших этот остаток.

## 5. Финансовая отчетность
- **Баланс (Форма 1):** Проверка равенства Активов (что имеем) и Пассивов (откуда взяли).
- **ОФР (Форма 2):** Детальный разбор прибылей и убытков по категориям расходов.

## 6. Настройки
- **Закрытый период:** Установите дату, до которой изменения в журнале запрещены.
- **Дата ввода остатков:** Укажите первый день начала учета в системе. Технический счет 0000 будет доступен для использования **только** на эту дату.
- **Фиксация остатков:** Переключатель, блокирующий редактирование проводок по вводу остатков после завершения инициализации.

## 7. Ввод начальных остатков (Account 0000)
Для того чтобы начать работу в системе с уже существующими остатками (деньги в банке, задолженности, основные средства), используется **Вспомогательный счет 0000** (Ввод начальных остатков).

### Процесс инициализации:
1.  **Настройка даты**: В разделе **Настройки** установите «Дату ввода остатков» (например, `01.01.2025`).
2.  **Ввод данных**: В **Журнале операций** создайте проводки на эту дату:
    - **Ввод остатка на счету**: Дт 5110 (Банк) / Кт 0000.
    - **Ввод долга поставщику**: Дт 0000 / Кт 6010.
    - **Ввод уставного капитала**: Дт 0000 / Кт 7410.
3.  **Контроль баланса**: В верхней части Журнала появится виджет **«Баланс ввода остатков (0000)»**. 
    - Виджет показывает общую сумму Дебета и Кредита по счету 0000.
    - Когда разница станет равной **нулю** (`✅ Баланс ок`), это означает, что сумма Активов совпала с Пассивами.
4.  **Фиксация**: Нажмите кнопку **«Зафиксировать остатки»** в виджете. Это действие заблокирует редактирование этих данных для предотвращения случайных искажений.

### Особенности отчетов:
- Счет **0000** не отображается в ОСВ, Балансе или ОФР.
- Начальные остатки из проводок с 0000 автоматически попадают в колонку **«Начальное сальдо»** в ОСВ на первый месяц работы.
- Все операции после даты ввода остатков формируют **«Обороты»**.
- Итоговое состояние счетов отображается в **«Конечном сальдо»**.
