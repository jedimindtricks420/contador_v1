# Многопользовательский и Мульти-организационный режим (Contador v2.0)

## 1. Концепция
Contador v2.0 поддерживает неограниченное число независимых юридических лиц (**Организаций**) в рамках одного пользовательского аккаунта. Это реализует полноценную Multi-tenancy архитектуру, позволяя бухгалтеру или владельцу бизнеса вести учет по нескольким компаниям без выхода из системы.

## 2. Изоляция данных
Данные каждой организации полностью изолированы друг от друга:
- **Финансы:** Транзакции одной компании никогда не пересекаются с другой.
- **Справочники:** Контрагенты и счета уникальны для каждого юрлица.
- **Настройки:** Каждая компания имеет свои даты закрытого периода и ввода остатков.

## 3. Механизм переключения (Organization Switcher)
Для удобства работы в левое боковое меню (Sidebar) интегрирован переключатель организаций:
- **Логика:** При выборе компании `is_active` флаг или `active_org_id` в модели `User` обновляется в базе данных.
- **Редирект:** После смены происходит перезагрузка страницы для сброса кеша Query Client и загрузки контекста новой организации.
- **UI:** Переключатель расположен в нижней части сайдбара рядом с информацией о профиле.

## 4. Управление организациями
В разделе **«Настройки» → «Мои компании»** пользователю доступны следующие действия:
1. **Список компаний:** Визуальный перечень всех принадлежащих пользователю юрлиц с указанием ИНН.
2. **Редактирование:** Возможность изменить название и ИНН организации в любой момент.
3. **Удаление:** Деструктивная операция. При удалении организации каскадно удаляются все связанные с ней транзакции, контрагенты, счета и локальные настройки.

## 5. Регистрация и Онбординг
- **Auth:** Система входа (`/login`) и регистрации (`/register`) заменяет старую Basic Auth.
- **Onboarding:** При отсутствии организаций у пользователя система автоматически запускает мастер настройки (Onboarding Flow), который помогает создать первую компанию и выбрать план счетов по отрасли.
