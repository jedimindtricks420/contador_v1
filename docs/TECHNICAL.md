# Техническое описание системы Contador v2.0

## Технологический стек
- **Framework:** Next.js 16+ (App Router, Turbopack)
- **Database:** PostgreSQL 16
- **ORM:** Prisma 6.2.1
- **Styling:** Tailwind CSS 4
- **Precision:** `Decimal.js` (все финансовые вычисления)
- **State Management:** TanStack Query (React Query) v5
- **Global UI State:** React Context (управление чатом и макетом)

## Архитектура Multi-tenancy
Система реализована по модели **изоляции на уровне БД (Shared Database, Shared Schema)**.

### Механизм доступа
1. **Пользователь (User):** Управляет неограниченным количеством организаций. Поле `active_org_id` в модели `User` определяет текущий контекст.
2. **Организация (Organization):** Все финансовые данные (`Transaction`, `Account`, `Counterparty`, `SystemSettings`) жестко привязаны к `organization_id`.
3. **Изоляция:** API-эндпоинты автоматически фильтруют данные через `getActiveOrganizationId()`, исключая доступ к данным чужих организаций.

### Глобальный макет (Enterprise Layout)
- **ClientLayout:** Динамически управляет рабочим пространством. При открытии ИИ-помощника макет плавно сдвигается влево (`pr-[400px]`), обеспечивая идеальную симметрию и центровку `mx-auto` контента.
- **UIContext:** Глобальное состояние `isChatOpen` позволяет любой части системы активировать адаптивный интерфейс.

## Логика Базы Данных (Prisma Models)
- **MasterAccount:** Эталонный план счетов НСБУ (~800 счетов). Служит базой для сединга всех организаций.
- **IndustryTemplate:** Отраслевой шаблон (например, «IT», «Розничная торговля»).
- **IndustryTemplateItem:** Связующая таблица для определения обязательных (`is_required`) счетов в конкретной отрасли.
- **Account:** Рабочий счет конкретной организации. Может быть активирован из шаблона или создан вручную.

## Финансовая логика (Accounting Logic)
Система построена на принципах **двойной записи (Double-entry bookkeeping)**.

### Валидация транзакций
- **Контроль периодов:** Запрет на редактирование или удаление проводок до даты `closed_period_date`.
- **Контроль остатков:** Операции со счетом 0000 разрешены только в дату `opening_balance_date`.
- **Защита счетов:** Нельзя деактивировать счет, если по нему есть ненулевой оборот или он помечен как обязательный в шаблоне организации.

### Формулы отчетов (ОСВ)
- **S1 (Начальное сальдо):** Сумма всех проводок (Дт - Кт) на дату ввода остатков.
- **Обороты:** Сумма транзакций за диапазон дат (исключая дату ввода остатков).
- **S2 (Конечное сальдо):** S1 + Обороты.

## Безопасность
- **Auth:** Полноценная регистрация и вход через `/login`. JWT-сессии.
- **Soft Delete:** Использование `is_deleted` для транзакций, обеспечивающее аудит и возможность восстановления.
- **Zod Validation:** Масштабная валидация всех входящих JSON-данных в API.
