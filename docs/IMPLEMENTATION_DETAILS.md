# Детали миграции и реализации

## Соответствие Excel
Система была спроектирована для точного соответствия вашей таблице учета:
- **Выручка 9010:** Аналог столбца "Доход" в Excel.
- **Расходы 94*:** Аналог операционных затрат (Маркетинг, Админ и т.д.).
- **ОСВ:** Заменяет сводные таблицы Excel для проверки остатков.

## Финансовая точность
В отличие от Excel, где возможны ошибки округления при `float` вычислениях, данная система использует библиотеку `Decimal.js` и тип `Decimal` в базе данных. Это гарантирует, что `0.1 + 0.2` всегда будет равно `0.3`, что критично для бухгалтерского баланса.

## Целостность данных
- **Double-Entry:** Вы не можете создать запись "из ниоткуда". Любое увеличение актива (Дебет) должно сопровождаться эквивалентным увеличением пассива или уменьшением другого актива (Кредит).
- **Технические счета:** Для ввода начальных остатков используется специальный счет `0000`, который выступает «корреспондирующим» для балансовых счетов на момент запуска системы.
- **Закрытый период:** В Excel любой может случайно изменить ячейку за прошлый год. Здесь это заблокировано программно как для текущих операций, так и для ввода остатков (через флаг `is_initial_balance_fixed`).

## Будущие расширения

3. **Автоматическое выставление счетов** (PDF генерация).
