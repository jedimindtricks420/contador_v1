# Модуль Онбординга и Отраслевые Планы Счетов (v2.0)

**Онбординг** в Contador v2.0 — это интеллектуальный процесс первичной настройки организации, который снижает порог входа для бухгалтера и предпринимателя.

## 1. Логика Онбординга (3 Шага)
Система реализована в виде FSM (Finite State Machine), которая сохраняет прогресс пользователя в поле `onboarding_state` модели `User`.

- **Шаг 1: `BASIC_INFO`**
  Пользователь вводит имя и ИНН компании. Создается запись в таблице `Organization`.
- **Шаг 2: `ACCOUNTING_SETTINGS`**
  Настройка учетной политики: дата ввода остатков (`opening_balance_date`) и дата закрытия периода.
- **Шаг 3: `SELECT_INDUSTRY`**
  Выбор отраслевого шаблона. Это критический шаг, на котором система копирует соответствующие записи из `MasterAccount` в `Account` организации.

## 2. 22 Отраслевых Шаблона
В Contador v2.0 предустановлены 22 шаблона, каждый из которых содержит набор обязательных (`is_required`) счетов:

1.  **IT & Software:** Разработка ПО, SaaS, Лицензии.
2.  **Розничная торговля:** Учет товаров (счет 2910), выручка.
3.  **Оптовая торговля:** Складской учет, дебиторка.
4.  **Производство:** Учет затрат (счет 2010), полуфабрикаты.
5.  **Гостиничный бизнес:** Услуги проживания.
6.  **Ресторанный бизнес:** Учет ингредиентов и блюд.
7.  **Логистика и Транспорт:** ГСМ, запчасти.
8.  **Строительство:** Незавершенное производство (счет 2080).
9.  **Сельское хозяйство:** Биологические активы.
10. **Образование:** Платные курсы.
11. **Медицинские услуги:** Аптека, услуги.
12. **Консалтинг:** Интеллектуальные услуги.
13. **Маркетинг и Реклама:** Диджитал-услуги.
14. **Электронная коммерция:** Маркетплейсы, доставка.
15. **Недвижимость и Аренда:** Долгосрочная аренда.
16. **Финансовые услуги:** Комиссии, проценты.
17. **Туризм:** Турагентства.
18. **Индустрия красоты (Салоны):** Расходные материалы.
19. **Авторемонт и Сервис:** Учет запчастей.
20. **Юридические услуги:** Гонорары, судебные издержки.
21. **Клининг и Бытовые услуги:** Химия, оборудование.
22. **Общий (Стандартный):** Универсальный план счетов НСБУ.

## 3. Механизм активации счетов
При выборе отрасли система:
1. Находит все счета, помеченные для этой отрасли в `IndustryTemplateItem`.
2. Создает копии этих счетов в таблице `Account` для данной организации.
3. Устанавливает статус `is_active: true` для обязательных счетов.
4. После завершения шага 3 `onboarding_state` пользователя переходит в `COMPLETED`.
