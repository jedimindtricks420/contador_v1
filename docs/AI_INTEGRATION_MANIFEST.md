# Манифест интеграции ИИ (AI Integration Manifest)

Этот документ предназначен для ознакомления ИИ-агентов с архитектурой и логикой системы **Contador v2.0**. Он содержит критическую информацию о создании объектов, безопасности и состоянии интерфейса.

---

## А. Логика транзакций и создания объектов

### Создание проводки (Transaction)
**Эндпоинт:** `POST /api/transactions`
**Путь:** `src/app/api/transactions/route.ts`

**Обязательные поля (JSON):**
- `date` (String): ISO дата (например, "2026-03-31").
- `period` (String): Формат "MM.YYYY" (например, "03.2026"). *Важно: ИИ должен вычислять это на основе даты.*
- `description` (String): Текстовое описание операции.
- `amount` (Number): Положительное число. В БД сохраняется как `Decimal`.
- `debit_id` (UUID): ID счета дебета из таблицы `Account`.
- `credit_id` (UUID): ID счета кредита из таблицы `Account`.
- `counterparty_id` (UUID, optional): ID контрагента.

**Валидация:**
- Запрещена запись в периоды ранее `SystemSettings.closed_period_date`.
- Специальный счет `0000` доступен только в дату `opening_balance_date`.

### Создание счета (Account)
**Эндпоинт:** `POST /api/accounts`
**Путь:** `src/app/api/accounts/route.ts`

**Логика связки:**
1. **Через шаблон:** Если передается `master_account_id`, данные копируются из `MasterAccount` (НСБУ).
2. **Кастомный:** Если передаются `code`, `name`, `type`, создается уникальный счет для организации.

## Б. Контекст и Безопасность

### Идентификация организации
**Функция:** `getActiveOrganizationId()`
**Путь:** `src/lib/context.ts`

**Механизм доступа:**
1. Извлекается `session` токен из Cookies.
2. Дешифруется (JWT/Secret) для получения `userId`.
3. В таблице `User` запрашивается поле `active_org_id`.
**Для ИИ:** Фильтрация всех данных (счета, проводки) ДОЛЖНА происходить строго по этому ID.

### Защита маршрутов
**Файл:** `middleware.ts`
- Все `/api/*` пути (кроме login/register) требуют валидную сессию.
- При отсутствии сессии API возвращает `401 Unauthorized`.

## В. Данные и Справочники

### Структура справочников для ИИ
Для корректного подбора счетов используйте:
- `GET /api/accounts`: Возвращает только активные счета текущей организации (ID, Code, Name).
- `GET /api/counterparties`: Список партнеров (ID, Name).

**Пример описания счета для промпта:**
```json
{
  "id": "uuid",
  "code": "1010",
  "name": "Денежные средства в кассе",
  "type": "ACTIVE"
}
```

### UI Чат (AIChat)
**Путь:** `src/components/AIChat.tsx`
- Сейчас это `fixed` панель справа.
- Для встраивания кнопок действий (например, «Создать транзакцию») нужно модифицировать секцию `Message Area`, добавив поддержку интерактивных компонентов в теле сообщения ИИ.

## Г. Глобальное состояние (UI)

### Состояние чата
**Контекст:** `UIContext`
**Путь:** `src/lib/ui-context.tsx`

**API состояния:**
- `isChatOpen`: Boolean. При `true` макет `ClientLayout` автоматически сжимает контент на 400px справа.
- `toggleChat()`: Функция переключения.

**Взаимодействие с интерфейсом:**
Чтобы чат «узнал» о выборе счета пользователем, рекомендуется добавить в `UIContext` поле `lastSelectedAccount`, которое будет обновляться в `SearchableSelect` и считываться в `AIChat` через `useEffect`.
